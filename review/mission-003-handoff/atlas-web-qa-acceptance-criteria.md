# Atlas Web QA Acceptance Criteria

## Approval Gates

### 1. Visual Quality
- Pass criteria:
  - Distinctive premium visual execution.
  - Strong typography hierarchy and spacing discipline.
  - Before/after proof is visually prominent above the fold.
- Fail if:
  - Generic template-like appearance.
  - Visual clutter or weak first-screen impact.

### 2. Brand Alignment
- Pass criteria:
  - Tone is calm, confident, outcome-focused.
  - Language reflects premium, trustworthy, commercially intelligent positioning.
- Fail if:
  - Hype language or futuristic clichés appear.

### 3. Messaging Accuracy
- Pass criteria:
  - Business outcomes and conversion improvement are primary message.
  - "evidence-backed" used where rationale language appears.
- Fail if:
  - Unsupported business-result guarantees or invented proof is present.

### 4. Demonstration Disclosure
- Pass criteria:
  - Demonstration work clearly labeled in proof areas and project cards.
  - Disclosure statement present and readable.
- Fail if:
  - Demonstrations are framed as prior client work.

### 5. Conversion Clarity
- Pass criteria:
  - CTA hierarchy is unambiguous.
  - Primary CTA is exactly: Request Your Free Website Audit.
- Fail if:
  - Competing primary CTAs or inconsistent phrasing.

### 6. Mobile Usability
- Pass criteria:
  - Navigation, proof modules, forms, and CTAs are fully usable on mobile.
  - Tap targets and spacing meet usability expectations.
- Fail if:
  - Weak mobile presentation or interaction breakage.

### 7. Accessibility
- Pass criteria:
  - Baseline WCAG 2.1 AA checks pass for contrast, focus, and form labeling.
- Fail if:
  - Critical accessibility blockers remain.

### 8. Form Behavior
- Pass criteria:
  - Form validates correctly and returns clear confirmation.
  - Error handling is clear and recoverable.
- Fail if:
  - Untested forms, broken submissions, or unclear errors.

### 9. Page Speed
- Pass criteria:
  - Pages load quickly on modern mobile networks.
  - Heavy assets optimized.
- Fail if:
  - Noticeable load friction from unoptimized media/scripts.

### 10. Navigation Integrity
- Pass criteria:
  - Required pages reachable with clear pathing.
  - Demonstration Projects discoverable before deep Process detail.
- Fail if:
  - Broken navigation or contradictory page hierarchy.

### 11. Broken Links
- Pass criteria:
  - No broken internal links or anchor failures.
- Fail if:
  - Any broken link in core journey.

### 12. Analytics Readiness
- Pass criteria:
  - Defined conversion events are wired for implementation.
- Fail if:
  - No event mapping for CTA and form actions.

### 13. SEO Readiness
- Pass criteria:
  - Page metadata, headings, and indexability controls complete.
- Fail if:
  - Missing titles/descriptions or structural SEO basics.

### 14. Unsupported Claims
- Pass criteria:
  - No fabricated metrics, customers, testimonials, awards, or guarantees.
- Fail if:
  - Any unsupported claim appears.

### 15. Privacy and Compliance
- Pass criteria:
  - Privacy and no-obligation language present where required.
- Fail if:
  - Missing privacy handling context for form capture.

### 16. Technical Handoff Quality
- Pass criteria:
  - Specialist handoff package complete and internally consistent.
- Fail if:
  - Placeholder content, unresolved critical facts, or contradictory instructions.

## Global Hard Fail Conditions
A website candidate is rejected automatically if it:
- Looks generic or template-derived.
- Uses invented proof.
- Exposes internal Atlas architecture.
- Overuses AI language.
- Has weak mobile presentation.
- Lacks clear CTA hierarchy.
- Represents demonstrations as client work.
- Makes guaranteed business-result claims.
- Contains placeholder content.
- Has untested forms or links.
