# Atlas Daily Operations SOP

Version: Sprint 21 Initial
Effective Date: 2026-07-10
Owner: Office of the CEO
Document Type: Standard Operating Procedure
Status: Active

## 1. Purpose
Define the daily operating rhythm for Atlas Studios so each production day is governed, evidence-based, and repeatable.

This SOP establishes a fixed executive cycle from morning selection through improvement intake without changing production architecture.

## 2. Scope
Applies to daily Atlas documentary operations only.

In scope:
- Executive decision rhythm and handoff order.
- Daily production mission selection and execution.
- Post-production governance decisions.
- Single-item improvement queue intake.

Out of scope:
- Architecture redesign.
- Specialist model changes.
- Atlas Academy modifications.
- Scheduling automation.

## 3. Non-Negotiable Constraints
1. Do not modify production architecture.
2. Do not add specialists.
3. Do not automate scheduling.
4. Do not modify Atlas Academy.
5. Focus on operating rhythm definition and execution discipline only.

## 4. Daily Operating Cycle
The Atlas Daily Operations cycle has six required phases and must run in order.

### Phase 1: Executive Morning Brief
Objective: Select today's production mission using latest operational evidence.

Required review inputs:
1. Previous benchmark package.
2. Executive Improvement Report from previous completed documentary.
3. Publish Decision from previous completed documentary.
4. Topic Evaluation shortlist for candidate missions.

Required decisions:
1. Confirm what was learned from previous benchmark.
2. Confirm whether prior publish decision implies risk carryover for today.
3. Select exactly one production mission for today.

Required outputs:
1. Morning Brief Record with selected mission ID.
2. Rationale for mission selection.
3. Deferred mission candidates list.

### Phase 2: Production Mission
Objective: Execute the selected documentary mission through the existing production pipeline.

Required actions:
1. Admit selected mission under existing runtime controls.
2. Execute documentary production under current architecture and specialist set.
3. Produce standard mission artifacts and release candidate package.

Required outputs:
1. Mission runtime result.
2. Release candidate artifacts.
3. Runtime diagnostics for executive review.

### Phase 3: Executive Producer Review
Objective: Evaluate release candidate quality and improvement opportunities.

Required actions:
1. Review release candidate package.
2. Generate Executive Improvement Report.
3. Ensure recommendations reference concrete evidence (for example, narrative beat IDs where relevant).

Required outputs:
1. Executive Improvement Report.
2. Ranked improvement recommendations.
3. Explicit note that recommendations are advisory until selected in Improvement Queue.

### Phase 4: Publish Decision
Objective: Produce the governance decision after Executive Producer Review.

Required actions:
1. Evaluate publish readiness from review evidence.
2. Issue one explicit decision outcome: PUBLISH, REVISE, or REJECT.
3. Record decision confidence and rationale.

Required outputs:
1. Publish Decision Report.
2. Decision rationale record.
3. If REVISE or REJECT, documented return conditions.

### Phase 5: CEO Review Checkpoint
Objective: Confirm executive control over day-end mission disposition.

Required actions:
1. Review Morning Brief intent versus actual mission outcome.
2. Review Executive Improvement Report and Publish Decision together.
3. Confirm go-forward posture for next day (continue pattern, tighten controls, or strategic redirect).

Required outputs:
1. CEO checkpoint note.
2. Confirmed executive posture for next operating day.

### Phase 6: Improvement Queue
Objective: Convert review recommendations into controlled improvement intake.

Required rules:
1. Select exactly one improvement candidate.
2. Defer all other recommendations.
3. Do not execute deferred recommendations in the same daily cycle.

Required actions:
1. Evaluate recommendation impact and urgency.
2. Choose one candidate with strongest expected executive value.
3. Record all non-selected items as deferred backlog.

Required outputs:
1. Improvement Queue Decision Record.
2. One selected improvement candidate.
3. Deferred recommendations register.

## 5. Decision Records Required Per Day
Every operating day must produce the following records:
1. Morning Brief Record.
2. Production Mission Result Summary.
3. Executive Improvement Report.
4. Publish Decision Report.
5. CEO Review Checkpoint Note.
6. Improvement Queue Decision Record.

A daily cycle is incomplete if any required record is missing.

## 6. Control Rules
1. The cycle is governance-first and evidence-first.
2. Exactly one mission is selected in the morning brief for that cycle.
3. Exactly one improvement candidate is selected in the improvement queue for that cycle.
4. All remaining recommendations are deferred, not discarded.
5. Publish Decision must exist before CEO checkpoint closure.
6. This SOP defines rhythm only and does not change technical architecture.

## 7. Standard Operating Procedure Status
This document is the Atlas Studios standard operating procedure for daily operations starting Sprint 21.

Any future revision must preserve the six-phase order unless explicitly superseded by CEO-approved operations governance.
